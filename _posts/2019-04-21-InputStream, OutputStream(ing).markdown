* What is stream? why do I need it?
A stream is defined as a 'sequence of data'. InputStream is used to read data from a source, and OutputStream is used to write data to a destination. So the code containing Java's i/o library, once compiled by JVM, somehow reads or writes things. Whenever there is a need for reading something, say a response body delivered via HTTP protocol, InputStream is used. Whenever I want to write something as a response either as a file or packet, I used the OutputStream.

* Kinds of i/o streams
The kinds of Java i/o streams can be subdivided into the following categories.
![img]({{ "/assets/img/iostreams.png" | absolute_url }})

- ByteStreams : All sorts of data including images, multimedia, characters and etc can be read and written.
- CharacterStreams: Only characters can be read and written.

I have seen that in many cases, 'BufferedStreams' are used. 
BufferedInputStream is not an abstract class, hence I can create an instance of it. When the bufferedStream reads with read(), it still returns one byte at a time, but it also reads ahead to fill a buffer. It makes most calls to read non-blocking, meaning that the retrieving process is not hindered by any other processes.
(https://coderanch.com/t/396675/java/Difference-BufferedInputStream-InputStream)
