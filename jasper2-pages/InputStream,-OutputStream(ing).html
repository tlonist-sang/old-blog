  • What is stream? why do I need it? A stream is defined as a ‘sequence of data’. InputStream is used to read data from a source, and OutputStream is used to write data to a destination. So the code containing Java’s i/o library, once compiled by JVM, somehow reads or writes things. Whenever there is a need for reading something, say a response body delivered via HTTP protocol, InputStream is used. Whenever I want to write something as a response either as a file or packet, I used the OutputStream.

  • Kinds of i/o streams The kinds of Java i/o streams can be subdivided into the following categories.

  • ByteStreams : All sorts of data including images, multimedia, characters and etc can be read and written.
  • CharacterStreams: Only characters can be read and written.

import java.io.FileInputStream;
public class FileInputStreamExam
{
    public static void main(String[] args)
    {
        // Byte unit file reading
        String filePath = "src/test.text"; // file path can be absolute or relative
        FileInputStream fileStream = null; 
        int i = 0; // variable for storing information of one byte
        try
        {
            fileStream = new FileInputStream(filePath);// making a FileStream
            
            do
            {
                i = fileStream.read(); 
                System.out.print((char) i);
            }
            while(i != -1);
            
        }
        catch (Exception e)
        {
            System.out.println("File I/O error!!" + e);
        }
        finally
        {
            try
            {
                fileStream.close();// closing file I/O
            }
            catch (Exception e)
            {
                System.out.println("Failed" + e);
            }
        }
    }
}

I have seen that in many cases, ‘BufferedStreams’ are used. BufferedInputStream is not an abstract class, hence I can create an instance of it. When the bufferedStream reads with read(), it still returns one byte at a time, but it also reads ahead to fill a buffer. It makes most calls to read non-blocking, meaning that the retrieving process is not hindered by any other processes. (https://coderanch.com/t/396675/java/Difference-BufferedInputStream-InputStream)

import java.io.BufferedInputStream;
import java.io.BufferedOutputStream;
import java.io.FileInputStream;
import java.io.FileOutputStream;
 
public class FileInputStreamExam
{
    public static void main(String[] args)
    {
        // Reading files in byte unit
        String filePath = "src/test.zip"; // object file
        FileInputStream inputStream = null; 
        FileOutputStream outputStream = null;
        BufferedInputStream bufferedInputStream = null; // BufferedStream
        BufferedOutputStream bufferedOutputStream = null;
 
        try
        {
            inputStream = new FileInputStream(filePath);// Instantiating FileInputStream
            bufferedInputStream = new BufferedInputStream(inputStream);// Instantiating FileOutStream
            
            outputStream = new FileOutputStream("src/test.zip");
            bufferedOutputStream = new BufferedOutputStream(outputStream);
 
 
            byte[] readBuffer = new byte[1024];
            while (bufferedInputStream.read(readBuffer, 0, readBuffer.length) != -1)
            {
                //writes to buffereOutputStream
                bufferedOutputStream.write(readBuffer);
            }
 
        }
        catch (Exception e)
        {
            System.out.println("File I/O error!!" + e);
        }
        finally
        {
            try
            {
                // Closing BufferedStreams
                bufferedInputStream.close();
                bufferedOutputStream.close();
            }
            catch (Exception e)
            {
                System.out.println("Failed" + e);
            }
        }
    }
}
